## 查看方法

### 方法一：浏览器直接打开
```
双击任意 .svg 文件 → 默认浏览器打开
```

### 方法二：在VS Code中预览
安装 SVG Preview 扩展后，直接在编辑器中预览。

### 方法三：转换为PNG
```bash
# 如果安装了 Inkscape：
inkscape 角色-沈暮雪.svg --export-type=png --export-dpi=300

# 如果安装了 rsvg-convert：
rsvg-convert -d 300 -p 300 角色-沈暮雪.svg -o 角色-沈暮雪.png
```

## 设计说明

### 角色 SVG 的意义

这些 SVG 不是最终美术——它们是**像素布局的精确底稿**。

每张图用 `<rect>` 标签精确控制每个像素方块的位置和颜色，可以直接作为：
1. **Aseprite 描摹参考**：在Aseprite中打开32×32画布，对照SVG布局逐格填色
2. **程序化生成基础**：SVG的rect坐标可以直接转换为游戏中的像素数据
3. **配色验证**：检查角色在实际背景色（深蓝底）中的可辨识度

### 色彩验证

所有概念图都使用深蓝底色 `#1a1a2e`——这是游戏中战斗场景最暗的背景色。

如果角色在这个背景下能清晰辨识——说明配色方案成立。

### 后续如何使用

| 步骤 | 操作 |
|------|------|
| 1 | 美术打开Aseprite，新建32×32画布 |
| 2 | 参照SVG底稿，逐格填色 |
| 3 | 添加动画帧（idle/walk/attack等） |
| 4 | 导出为精灵表（Sprite Sheet） |
| 5 | 导入Unity/Godot使用 |

## 后续可扩展

如果需要更多概念图，可以继续用SVG生成：
