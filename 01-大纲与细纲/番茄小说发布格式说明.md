# 番茄小说发布格式说明

> 本文件用于将 markdown 正文转换为番茄小说可发布的纯文本格式。

---

## 一、番茄平台硬性要求

| 项目 | 要求 |
|------|------|
| 单章字数 | 最低 2000 字（签约线），建议 3000-4000 字 |
| 文件格式 | 纯文本，不支持 markdown |
| 章节标题 | "第一章 标题" 或 "第1章 标题"，不加 # 号 |
| 段落分隔 | 段落之间空一行（回车×1 即可，不要空两行） |
| 对话标点 | 中文引号 ""，不要用英文引号 |
| 分隔符 | 不建议使用 ---、*** 等符号分隔线 |
| 表情/符号 | 不要用 emoji、★、✓ 等特殊符号 |

## 二、当前文件需要做的转换

### 2.1 章节标题
```
❌ 错误（当前格式）：
# 第一章 死而复生

✅ 正确（番茄格式）：
第一章 死而复生
```

### 2.2 加粗/斜体
```
❌ 错误：**他笑了。**
✅ 正确：他笑了。

❌ 错误：*她低声说*
✅ 正确：她低声说
```
→ 所有 ** 和 * 标记全部删除。

### 2.3 表格/列表
```
❌ 错误（markdown 表格）：
| 项目 | 数量 |

✅ 正确：直接用正文叙述，如"大米50斤×20袋"
```
→ 正文中的表格改为自然段落叙述。

### 2.4 分隔线
```
❌ 错误：---
✅ 正确：（直接空行过渡，不加任何符号）
```

### 2.5 代码块/系统提示
```
❌ 错误：【检测到宿主...】用代码块包裹

✅ 正确：【检测到宿主...】直接作为正文段落
   可以用 【】 或 「」 包裹系统提示，番茄支持
```

## 三、建议的发布流程

1. 在 markdown 文件中完成创作和修改
2. 定稿后，用以下命令批量转换：
   - 删除所有 # 号
   - 删除所有 ** 和 *
   - 删除所有 --- 分隔线
   - 检查特殊符号
3. 复制纯文本到番茄作家后台发布

## 四、快速转换命令

```bash
# 将 markdown 正文转为纯文本（在项目目录下执行）
sed -e 's/^#* //g' -e 's/\*\*//g' -e 's/\*//g' -e '/^---$/d' 03-正文/第001章-死而复生.md > 发布版/第001章.txt
```

## 五、番茄平台写作注意事项

1. **黄金前三章**：番茄读者免费阅读前三章，必须在 3000 字内抓住读者。第一章开头 500 字必须有爆点。
2. **每章结尾留钩子**：番茄按章分发，章尾悬念决定追读率。
3. **避免大段描写**：手机阅读，段落不宜超过 4-5 行。频繁分段提升阅读节奏。
4. **对话占比高**：番茄读者偏好对话密集的快节奏，对话占比建议 40% 以上。
5. **书名建议**：番茄书名要直白吸睛，建议格式"末世重生+空间+女主"。例如：
   - 《末世重生：我有灵泉空间》
   - 《重生末世前，我疯狂囤货十亿》
   - 《末世天灾：空间女皇不圣母》
